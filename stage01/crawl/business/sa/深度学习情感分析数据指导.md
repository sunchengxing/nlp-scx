# 深度学习情感分析 — 数据爬取与建模指导

## 一、定位

| 项目 | 说明 |
|------|------|
| **优先级** | 主要任务 |
| **目标** | 爬取股吧文本数据，训练深度学习情感分类模型，实现舆情分析 |
| **核心** | 文本 → 预训练模型微调 → 情感分类（看多/看空/中性） |

---

## 二、深度学习需要爬什么数据

深度学习不需要你手工造特征，它需要的是：**足够多的原始文本 + 标签**。

### 爬取清单

| 数据 | 字段 | 说明 |
|------|------|------|
| **股吧帖子** | title, content, post_time, stock_code | 核心训练数据，模型从文本中学情感 |
| **帖子元数据** | read_count, comment_count, like_count | 可选，辅助筛选高质量样本 |

### 与传统ML的关键区别

| | 传统ML | 深度学习 |
|---|--------|---------|
| **需要什么数据** | 结构化字段（数值、类别） | 原始文本 |
| **特征从哪来** | 你手工造 | 模型自动学 |
| **content 重要吗** | 不重要（要数值） | **非常重要**（文本越多信号越强） |
| **数据量需求** | 几千条够 | 越多越好，1万起步 |

### 所以深度学习必须爬帖子正文（content）

标题只有十几个字，信息太少。正文才是情感信号的主要载体：

```
标题："明天怎么看"                    → 没有任何情感信号
正文："大盘量能不足，短期大概率回调，
      建议减仓观望，等缩量企稳再进场"  → 明确看空信号
```

## 三、数据爬取方案

### 3.1 爬取范围

| 参数 | 建议 | 理由 |
|------|------|------|
| **标的** | 2~3只热门股（茅台600519、宁德300750、比亚迪002594） | 多行业覆盖，验证泛化性 |
| **时间** | 最近3~6个月 | 够拿 2K 条即可，不需要太长 |
| **每只帖数** | 目标 700~1000 条 | 3只 × 700 ≈ 2.1K |
| **总量目标** | **约 2K 条** | 作为测试集，AI标注+人工审核 |

### 3.2 爬取字段设计

```json
{
  "stock_code": "600519",
  "stock_name": "贵州茅台",
  "title": "明天突破2000",
  "content": "放量突破压力位，技术面走好...",
  "post_time": "2025-06-10 14:32:00",
  "read_count": 12580,
  "comment_count": 43,
  "like_count": 12,
  "author": "股友xxx",
  "source": "eastmoney_guba"
}
```

### 3.3 数据质量控制

爬的时候就要做初步筛选，减少后续清洗负担：

- [ ] **过滤广告帖** — 含"加群""V信""荐股大师""牛股推荐"
- [ ] **过滤超短帖** — 正文 < 10 字的，信号太弱
- [ ] **过滤纯转载** — 只有链接没有原创内容的
- [ ] **去重** — 同一作者同日同标题只保留一条

## 四、标签策略（已确定方案）

### 最终方案：AI 标注 + 人工审核

```
训练集: HikasaHana 全量 10,650 条（train 8,520 + test 2,130 合并，人工标注）
测试集: 自己爬取 2K 条 → AI 标注 → 人工审核修正
```

### 标签定义（沿用 HikasaHana 规范，金融领域特有）

| label | 含义 | 说明 |
|-------|------|------|
| 0 | **negative（看空）** | 对股票上涨持消极态度 / 对股票下跌持积极态度 |
| 1 | **positive（看多）** | 对股票上涨持积极态度 / 对股票下跌持消极态度 |
| 2 | **neutral（中性）** | 无主观倾向的提问、与走势无关的内容、多空摇摆的观点 |

> ⚠️ 这里的 positive/negative 是"看涨/看跌"，不是"开心/难过"。
> "暴跌好！" → 通用情感 positive，但金融情感 **negative（看空）**。

### AI 标注 Prompt 设计要点

给 AI 标注时，prompt 必须包含：
1. 明确的标签定义（看多/看空/中性，不是开心/难过）
2. 标注示例（正例 + 反例 + 边界案例）
3. 要求 AI 输出结构化结果（label + confidence + reasoning）
4. confidence < 0.7 的标记为"待人工重点审核"

### 人工审核流程

```
AI 标注 2K 条
  ├── confidence ≥ 0.7 (~1500条):  抽检 10%（~150条）
  └── confidence < 0.7 (~500条):   全量人工审核
  → 修正错误标签
  → 最终得到高质量 2K 测试集
```

### 为什么爬 2K 就够

- HikasaHana 原测试集 2,130 条，统计上 3 分类每类约 700 条，已够稳
- 2K 条测试集与原数据集可比，结果有参考价值
- AI 标注 + 人工审核 2K 条，大约 **1~2 小时**可完成

### 为什么把原 test 并入训练

- HikasaHana 的 test 也是人工标注的，质量一样
- 训练数据从 8,520 → 10,650，多 25%，对小数据集有实际帮助
- 用自己爬的数据做测试，避免了"训练集见过的数据当测试"的问题
- 测试集来自真实爬取，更能反映模型在实际场景的表现

## 五、数据清洗与预处理

### 5.1 文本清洗（喂给模型前必须做）

| 步骤 | 操作 | 原因 |
|------|------|------|
| 去HTML标签 | `re.sub(r'<[^>]+>', '', text)` | 爬下来的带HTML残留 |
| 去URL | 去除 http 开头的链接 | 无情感信号 |
| 去@用户 | 去除 @xxx 格式 | 无情感信号 |
| 统一标点 | 全角→半角，连续标点合并 | 减少噪声token |
| 去除纯数字帖 | 正文全是数字的帖子 | 无语义 |
| 保留！？ | **保留感叹号和问号** | 它们有情感信号 |

### 5.2 预处理

| 步骤 | 说明 |
|------|------|
| 分词 | 不需要！BERT 类模型自带 tokenizer，会自动处理 |
| 停用词 | 不需要！预训练模型自己学哪些词重要 |
| 文本拼接 | title + [SEP] + content，拼成一条输入 |
| 长度截断 | 看 length 分布决定 max_length，一般 128 或 256 |

### 5.3 数据集划分

```
全量数据
  ├── train: 80%
  ├── val:   10%
  └── test:  10%

注意：按时间划分，不要随机划分！
  train 用前80%时间段的数据
  test  用后20%时间段的数据
  这样才能验证模型对"未来数据"的泛化能力
```

> ⚠️ 时间序列数据随机划分 = 数据泄露，test 上看着好实际没用。

## 七、评估

### 7.1 核心指标

| 指标 | 为什么看 |
|------|---------|
| **F1-macro** | 三分类各标签均衡性 |
| **混淆矩阵** | 看多/看空/中性之间的误判模式 |
| **Per-class F1** | 每个类别的 F1，看哪个类最差 |
| **accuracy** | 参考，不作为核心指标 |

### 7.2 对比实验

| 对比维度 | 具体对比 |
|---------|---------|
| 模型对比 | bert-wwm vs roberta-wwm vs macbert |
| 输入对比 | title only vs title+content vs content only |
| 标签对比 | 3分类 vs 2分类（去中性） |
| 数据量对比 | 5千条 vs 1万条 vs 3万条，看数据量对效果的影响 |

### 7.3 过拟合检查

- [ ] train loss vs val loss 曲线是否发散

## 九、执行步骤总结

```
1. 爬数据：2~3只热门股，每只700条，含 title + content + 评论，总量约2K
2. 清洗：去广告、去HTML、去短帖
3. AI 标注：用 AI 对 2K 条打标签（0/1/2），输出 label + confidence + reasoning
4. 人工审核：低置信度全审，高置信度抽检 → 最终高质量 2K 测试集
5. 训练：用 HikasaHana 全量 10,650 条训练 baseline（bert-wwm-ext）
6. 评估：用 2K 测试集评估 F1-macro、混淆矩阵
7. 对比实验：换模型、换输入(title only vs title+content)
8. 选出最优模型，保存权重
```

## 十、与 HikasaHana 数据集的关系（已确定方案）

| | HikasaHana 数据集 | 你自己爬的数据 |
|---|-------------------|---------------|
| **用途** | **训练集**（全量 10,650 条） | **测试集**（~2K 条） |
| **标签** | 人工标注 | AI标注 + 人工审核 |
| **优点** | 已标注，开箱即用 | 多股票、有正文，真实场景验证 |
| **缺点** | 仅上证50ETF、仅标题 | 需要标注流程 |

**确定方案**：
1. HikasaHana 的 train(8,520) + test(2,130) **全部作为训练集** = 10,650 条
2. 自己爬 2K 条 → AI 标注 → 人工审核 → **作为测试集**
3. 训练集和测试集标签定义一致（都是帖子情感，不是行情结果）
4. 测试集来自真实爬取，反映模型实际表现

## 十一、关键风险

| 风险 | 说明 | 应对 |
|------|------|------|
| 数据量不足 | HikasaHana 仅1万条 | 合并train+test = 10,650条，够微调 |
| 标签噪声 | AI标注有错误 | 人工审核低置信度样本，抽检高置信度样本 |
| 过拟合 | 小数据+大模型=容易过拟合 | early stopping + dropout + 数据增强 |
| 领域偏移 | 训练(上证50ETF标题) vs 测试(多股票标题+正文) | 正好验证泛化能力，差距大说明需要更多训练数据 |
| 测试集仅有标题 | 和 HikasaHana 一样只有标题 | **必须爬正文**，测试时用 title+content 评估 |
