# 传统机器学习全流程实践指导

## 一、定位

| 项目 | 说明 |
|------|------|
| **优先级** | 次要（练手复习，非主线任务） |
| **目的** | 亲身走一遍传统ML全链路：原始数据 → 清洗 → 特征工程 → 建模 → 评估 |
| **核心价值** | 练特征工程能力，理解"数据决定上限，模型逼近上限" |

---

## 二、需要爬什么数据

传统ML需要**你能手工造特征**的数据，纯文本丢进去没有意义。

### 爬取清单

| 数据 | 来源 | 字段 |
|------|------|------|
| **股吧帖子** | 东方财富股吧 | title, post_time, read_count, comment_count, like_count |
| **日线行情** | 东方财富/AKShare | date, open, high, low, close, volume, turnover, pct_chg |

### 爬取范围

- **标的**：1只热门股（如贵州茅台）
- **时间**：最近6个月
- **量级**：帖子几千条 + 行情120条左右，刚好够练

### 为什么不爬帖子正文？

- 传统ML练的是**数值型特征工程**，正文处理要分词→向量，那已经是深度学习的活
- 标题 + 元数据（阅读数、评论数、发帖时间）+ 行情数据，字段足够你造特征了

---

## 三、Label 定义

**从行情数据自动生成，不依赖人工标注：**

```
次日涨跌幅 next_pct_chg:
  > +1%  → 1（涨）
  < -1%  → 0（跌）
  其余   → 2（平）
```

> 预测目标：**根据当日舆情特征 → 预测次日涨跌**。这比"判断单条帖子情感"更有实际意义。

## 四、全流程路线

```
原始数据（粗糙）
    │
    ▼
┌──────────┐
│  数据清洗  │  去重、去广告、去空值、时间对齐
└──────────┘
    │
    ▼
┌──────────┐
│  特征工程  │  从原始字段手工造特征 ← 这一步是传统ML的核心
└──────────┘
    │
    ▼
┌──────────┐
│  建模训练  │  LR → RF → XGBoost，逐步对比
└──────────┘
    │
    ▼
┌──────────┐
│  评估调优  │  混淆矩阵、特征重要性、调参
└──────────┘
```

### 4.1 数据清洗

- [ ] 去除重复帖（同一标题同一时间）
- [ ] 去除广告/推广帖（含"加群""荐股"等关键词）
- [ ] 处理缺失值（comment_count 为空填 0 还是删？）
- [ ] 时间对齐：帖子归属到对应交易日（盘后发的算次日？盘前算当日？）
- [ ] 合并帖子表和行情表（按日期 join）

### 4.2 特征工程（核心练习）

**文本侧特征**（从 title 提取）：

| 特征名 | 造法 | 直觉 |
|--------|------|------|
| title_len | `len(title)` | 标题长度，长帖可能更情绪化 |
| exclaim_count | `title.count('！') + title.count('!')` | 感叹号多 = 情绪强烈 |
| question_count | `title.count('？') + title.count('?')` | 问号多 = 犹豫/困惑 |
| pos_word_count | 正面词词典匹配计数 | 看多信号 |
| neg_word_count | 负面词词典匹配计数 | 看空信号 |
| sentiment_ratio | pos / (pos + neg + 1) | 情感倾向比例 |

> 正面词词典：涨、多、牛、冲、突破、加仓...
> 负面词词典：跌、空、熊、割、跑、崩盘、清仓...

**元数据特征**（从帖子统计量）：

| 特征名 | 造法 | 直觉 |
|--------|------|------|
| post_count | 当日发帖总数 | 讨论热度 |
| avg_read | 当日平均阅读数 | 关注度 |
| avg_comment | 当日平均评论数 | 争论程度 |
| read_comment_ratio | avg_read / (avg_comment + 1) | 旁观vs参与比 |
| post_count_chg | 当日帖数 / 前日帖数 - 1 | 热度变化率 |

**行情侧特征**（从日线行情）：

| 特征名 | 造法 | 直觉 |
|--------|------|------|
| pct_chg | 当日涨跌幅 | 最直接的行情信号 |
| amplitude | (high - low) / pre_close | 波动率 |
| volume_ratio | 当日量 / 5日均量 | 量比 |
| turnover | 换手率 | 活跃度 |
| ma5_slope | 5日均线斜率 | 趋势方向 |

**交叉特征**（最有价值的部分）：

| 特征名 | 造法 | 直觉 |
|--------|------|------|
| panic_index | pct_chg < -2 且 post_count 5日最高 | 恐慌指标 |
| greed_index | pct_chg > 2 且 pos_word_count 5日最高 | 贪婪指标 |
| sentiment_divergence | 股价跌但 pos > neg | 多空分歧 |

### 4.3 建模

**按顺序跑，每一步对比上一步的提升：**

| 步骤 | 模型 | 目的 |
|------|------|------|
| 1 | 逻辑回归 | baseline，特征重要性可解释 |
| 2 | 随机森林 | 非线性能力，看特征重要性排序 |
| 3 | XGBoost | 工业界标配，练调参 |

**评估指标：**
- accuracy（参考）
- F1-macro（三分类均衡性）
- 混淆矩阵（看哪类最容易错）
- 特征重要性排序（理解哪些特征真有用）

### 4.4 调优练习

- [ ] 删掉重要性最低的3个特征，看效果变化
- [ ] XGBoost 调参：max_depth、learning_rate、n_estimators
- [ ] 尝试归一化 vs 不归一化对 LR 的影响
- [ ] 交叉验证 vs hold-out，结果是否一致

## 五、时间分配建议

| 阶段 | 预计耗时 | 说明 |
|------|---------|------|
| 爬数据 | 不需要你管 | 工程化的事 |
| 数据清洗 | 1小时 | 主要是对齐和去噪 |
| 特征工程 | 2~3小时 | **重点投入**，造特征比跑模型有价值 |
| 建模评估 | 1小时 | sklearn 几行代码，重点是对比分析 |
| 调优复盘 | 1小时 | 特征重要性分析，理解什么特征真有用 |

**总计约半天，练完收到。**

---

## 六、关键收获

做完这个任务，你应该能回答：

1. 特征工程到底在干什么？ — 从原始数据造出模型能用的信号
2. 什么特征有用什么没用？ — 特征重要性会告诉你
3. 传统ML和深度学习的分界在哪？ — 你手动造特征 vs 模型自动学特征
4. "数据决定上限"怎么理解？ — 垃圾特征喂什么模型都烂
