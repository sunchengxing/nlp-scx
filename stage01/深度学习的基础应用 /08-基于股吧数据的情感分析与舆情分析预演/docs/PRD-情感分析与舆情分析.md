## 一、项目概述

### 1.1 项目背景

东方财富股吧是国内最大的股票讨论社区之一，每日产生大量帖子与评论。这些文本数据蕴含了投资者对股票走势的态度倾向——看多、看空或中性。通过 NLP 技术对这些文本进行情感分类，并叠加浏览量/点赞数等权重，可以构建有意义的舆情指标，辅助投资决策。

### 1.2 项目目标

| 目标 | 说明 |
|------|------|
| **核心目标** | 训练金融情感分类模型，对股吧帖子标题进行看多/看空/中性三分类 |
| **延伸目标** | 叠加浏览量、点赞数权重，构建加权舆情指标 |
| **预演性质** | 本项目为技术预演，验证流程可行性，非生产级部署 |

### 1.3 项目范围

| 包含 | 不包含 |
|------|--------|
| HF baseline 数据集训练 | 生产级 API 服务部署 |
| 本地帖子数据验证 | 实时流式舆情监控 |
| 加权舆情指标计算 | 前端可视化大盘 |
| 评论情感模型（阶段3） | 多股吧扩展 |

## 二、术语定义

| 术语 | 定义 |
|------|------|
| **情感分析** | 对单条文本判断情感倾向（看多/看空/中性） |
| **舆情分析** | 在情感分析基础上，叠加权重和时序，判断群体态度趋势 |
| **看多 (positive)** | 对股票上涨持积极态度，label=1 |
| **看空 (negative)** | 对股票上涨持消极态度/对下跌持积极态度，label=0 |
| **中性 (neutral)** | 无主观倾向的提问、与走势无关的内容，label=2 |
| **权重** | 基于浏览量、点赞数等量化文本影响力的系数 |
| **Baseline** | 基于公开数据集训练的基准模型 |
| **FinBERT** | 金融领域专用预训练 BERT 模型 |

> ⚠️ **金融情感 ≠ 通用情感**："暴跌好！"在通用情感里是 positive，金融情感里是 **negative（看空）**。

## 三、数据源

### 3.1 Baseline 训练数据

| 项目 | 详情 |
|------|------|
| 数据集名称 | `HikasaHana/eastmoney_guba_title` |
| 来源 | 东方财富华夏上证50ETF股吧帖子标题 |
| 地址 | https://huggingface.co/datasets/HikasaHana/eastmoney_guba_title |
| 规模 | 10,646 条（train: 8,520 / test: 2,130） |
| 字段 | `title`（string, 1-79字）+ `label`（int, 0/1/2） |
| 标注方式 | 人工标注，三元分类 |
| 注意事项 | 原标注者自述偶有困惑，需抽样校验 |

### 3.2 本地验证数据

| 项目 | 详情 |
|------|------|
| 数据库 | MySQL @ 138.201.173.21:3360 / crawlee_data |
| 帖子表 | scx_guba_post（743条），含 title + read_count + comment_count |
| 评论表 | scx_guba_comment（3,337条），含 content + like_count |
| 用途 | 帖子 title 做验证 + 舆情计算；评论数据留给阶段3 |

### 3.3 数据字段映射

| HF baseline | 本地帖子表 | 匹配关系 |
|-------------|-----------|---------|
| title | scx_guba_post.title | ✅ 完全匹配 |
| label | scx_guba_post.label（待预测） | 模型填充 |
| — | scx_guba_post.read_count | HF 无，本地有，用于权重 |
| — | scx_guba_post.comment_count | HF 无，本地有，用于权重 |

## 四、功能需求

### 4.1 阶段划分

```
阶段1：情感分类模型训练          阶段2：舆情指标构建          阶段3：评论情感扩展
┌──────────────────┐        ┌──────────────────┐        ┌──────────────────┐
│ HF baseline 数据  │        │ 本地帖子数据       │        │ 本地评论数据       │

│ 预训练模型微调     │        │ 加权舆情指标计算   │        │ 训练评论情感模型   │
│ 评估 + 调参       │        │ 舆情趋势分析      │        │ 补充舆情维度       │
└──────────────────┘        └──────────────────┘        └──────────────────┘
```

### 4.2 阶段1：情感分类模型训练

#### FR-1.1 数据加载与探索

| 编号 | 需求 | 验收标准 |
|------|------|---------|
| FR-1.1.1 | 加载 HF 数据集 | 成功加载 train/test split |
| FR-1.1.2 | 标签分布统计 | 输出各标签数量与占比 |
| FR-1.1.3 | 文本长度分布 | 输出长度统计（min/max/mean/median） |
| FR-1.1.4 | 抽样校验 | 随机抽取 50 条人工检查标注质量 |

#### FR-1.2 数据预处理

| 编号 | 需求 | 验收标准 |
|------|------|---------|
| FR-1.2.1 | 数据清洗 | 去除纯表情、纯标点、重复帖、广告帖 |
| FR-1.2.2 | Tokenization | 使用预训练模型配套 tokenizer 处理 |
| FR-1.2.3 | 数据集划分 | 从 train 中切出 10% 作为 validation |
| FR-1.2.4 | max_length 确定 | 根据长度分布确定，建议 32-64 |

#### FR-1.3 模型训练

| 编号 | 需求 | 验收标准 |
|------|------|---------|
| FR-1.3.1 | 选定预训练模型 | 至少对比 2 个模型（chinese-bert-wwm-ext + FinBERT） |
| FR-1.3.2 | 微调训练 | 使用 HuggingFace Trainer 完成训练 |
| FR-1.3.3 | 超参数配置 | epochs: 3-5, lr: 2e-5, batch_size: 16-32 |
| FR-1.3.4 | 早停机制 | validation loss 连续 2 轮不降则停止 |
| FR-1.3.5 | 模型保存 | 保存最优 checkpoint + 最终模型 |

#### FR-1.4 模型评估

| 编号 | 需求 | 验收标准 |
|------|------|---------|
| FR-1.4.1 | 准确率 | Accuracy ≥ 75%（baseline 参考） |
| FR-1.4.2 | F1-macro | 各标签 F1 均值 ≥ 0.70 |
| FR-1.4.3 | 混淆矩阵 | 输出三分类混淆矩阵，分析误判模式 |
| FR-1.4.4 | 看空精确率 | Precision(negative) ≥ 0.75（漏判看空风险大） |
| FR-1.4.5 | 看空召回率 | Recall(negative) ≥ 0.70 |

### 4.3 阶段2：舆情指标构建

#### FR-2.1 本地数据预测

| 编号 | 需求 | 验收标准 |
|------|------|---------|
| FR-2.1.1 | 从 DB 读取帖子 | 读取 scx_guba_post 全部 title + read_count + comment_count |
| FR-2.1.2 | 批量情感预测 | 模型对每条 title 预测 label + probability |
| FR-2.1.3 | 写回 DB | 将预测结果写入 scx_guba_post.label + confidence |

#### FR-2.2 加权舆情计算

| 编号 | 需求 | 验收标准 |
|------|------|---------|
| FR-2.2.1 | 权重定义 | weight = log(1 + read_count) × (1 + comment_count) |
| FR-2.2.2 | 舆情分数 | sentiment_score = Σ(label × weight) / Σ(weight) |
| FR-2.2.3 | 按日汇总 | 输出每日舆情分数 |
| FR-2.2.4 | 按股汇总 | 输出每只股票的舆情分数 |

### 4.4 阶段3：评论情感扩展（后续）

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-3.1 | 人工标注 200-500 条评论 | P2 |
| FR-3.2 | 训练评论情感模型 | P2 |
| FR-3.3 | 评论舆情指标补充 | P3 |

## 八、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 数据量偏少（8,520条） | 过拟合 | 早停 + weight_decay + 数据增强 |
| 标注质量存疑 | 模型学到错误模式 | 抽样校验 + 清洗 |
| 领域狭窄（仅上证50ETF） | 泛化能力差 | 后续补充其他股吧数据 |
| 评论与标题风格不同 | 阶段3效果打折 | 单独标注 + 训练评论模型 |
| 金融情感定义反直觉 | 误用通用情感模型 | 必须用金融标注数据训练 |
