# 基础 RNN 语言模型实验分析报告

> 实验日期：2026-06-15 ~ 2026-06-16
> 数据集：WikiText-2 (wikitext-2-raw-v1)
> 模型：手写基础 RNN（无 LSTM/GRU 门控）

---

## 一、实验概述

本实验从零搭建了一个基础 RNN 语言模型，在 WikiText-2 数据集上进行训练和评估。完整经历了数据处理、模型搭建、训练调优的完整流程。

---

## 二、数据处理流程

### 2.1 分词

- **方案选择**：正则分词 `r'\w+|[^\w\s]'`
- **选择理由**：英文语料，jieba 不适用；BPE 过重不适合入门
- **效果**：将标点单独拆分（如 `hello,` → `hello`, `,`），模型能学到标点规律

### 2.2 低频词过滤

- **阈值**：词频 < 5 替换为 `<unk>`
- **效果**：词表从 75,975 压缩到 ~22,000
- **影响**：
  - Embedding 层参数减少 71%
  - Output 层参数减少 71%
  - 模型体积从 115M → 35M
  - 训练速度提升约 3-5 倍

### 2.3 数据组织

- **reshape 方式**：`(bs, -1)` 即 75 条连续流
- **滑动窗口**：seq_len=20，每次取 20 个 token
- **错位切片**：x = data[:, :-1]，y = data[:, 1:]

## 三、模型架构

### 3.1 网络结构

```
Input (bs, seq_len)
  ↓
Embedding(vocab_size, 128)     → (bs, seq_len, 128)
  ↓ 逐时间步
x_linear(128, 256)             → (bs, 256)
h_linear(256, 256)             → (bs, 256)
  ↓ 相加 + tanh
h_t = tanh(W_xh·x_t + W_hh·h_{t-1})
  ↓
output_linear(256, vocab_size) → (bs, vocab_size)
```

### 3.2 参数量

| 层 | 参数量 |
|----|--------|
| Embedding | 22,000 × 128 = 2,816,000 |
| x_linear | 128 × 256 + 256 = 33,024 |
| h_linear | 256 × 256 + 256 = 65,792 |
| output_linear | 256 × 22,000 + 22,000 = 5,634,000 |
| **总计** | **~8,548,816** |

## 四、训练配置与结果

### 4.1 最终配置

| 参数 | 值 |
|------|-----|
| 优化器 | Adam (lr=0.001) |
| 损失函数 | CrossEntropyLoss |
| 梯度裁剪 | clip_grad_norm_=0.25 |
| batch_size | 75 |
| seq_len | 20 |
| epochs | 100 |

### 4.2 训练曲线

| Epoch | 平均 Loss | 每 Token Loss | Perplexity |
|-------|----------|--------------|------------|
| 1 | 117.11 | 5.86 | 350 |
| 5 | ~90 | ~4.50 | ~90 |
| 15 | 75.39 | 3.77 | 43 |
| 32 | 66.62 | 3.33 | 28 |
| 49 | 64.19 | 3.21 | 25 |
| 82 | 59.64 | 2.98 | 20 |
| 100 | 59.19 | 2.96 | **19** |

### 4.3 Perplexity 解读

```
Perplexity = exp(单 token loss)

随机猜测:  exp(ln(22000)) = 22,000   → 完全盲猜
当前模型:  exp(2.96)      = 19        → 每次约在 19 个词中选
理想模型:  exp(0)         = 1         → 完全确定（不可能）
```

**从 22,000 种可能缩小到 19 种，模型已经有效。**

## 五、基础 RNN 的瓶颈分析

### 5.1 梯度消失问题

基础 RNN 的梯度回传公式：

```
∂L/∂h_1 = ∂L/∂h_T × ∏(t=2 to T) [diag(σ') × W_hh]
```

每一步都要乘一个小于 1 的数（tanh 导数 ≤ 1），乘 20 次后梯度就很小了。

**实际表现**：模型无法学习超过 20 步的远距离依赖。

### 5.2 收敛天花板

从训练曲线可见：
- **前 15 轮**：loss 快速下降（117→75），模型在学习高频词和简单语法
- **15-50 轮**：缓慢下降（75→64），模型在学习局部搭配
- **50-100 轮**：几乎不动（64→59），基础 RNN 已达架构极限

### 5.3 为什么更多轮次没用

| 问题 | 表现 | 根因 |
|------|------|------|
| 梯度消失 | 远距离信息传不到 | tanh 导数 ≤ 1，连乘衰减 |
| 记忆容量 | 信息存不下 | hidden_dim=256 有限 |
| 优化困难 | loss 震荡不降 | 非凸优化 + 长期依赖冲突 |

**这不是训练不够的问题，是架构能力的问题。**

## 六、优化措施效果回顾

| 优化 | 加入前 | 加入后 | 提升幅度 |
|------|--------|--------|---------|
| 低频词过滤 (<5 → unk) | 词表 76K | 词表 22K | 体积 -70%，速度 +3x |
| SGD → Adam | lr=0.01 | lr=0.001 | 收敛速度显著提升 |
| 梯度裁剪 | 训练不稳定 | clip=0.25 | 防止梯度爆炸 |
| 增加轮次 (2→100) | loss=5.86 | loss=2.96 | -50%，但后期收益递减 |

## 七、结论与下一步

### 7.1 核心结论

1. **基础 RNN 能工作，但有天花板**——perplexity 19 是这个架构的合理水平
2. **低频词过滤是最有效的优化**——投入小、收益大
3. **优化器选择比轮次更重要**——Adam 比 SGD+更多轮次效果更好
4. **架构决定上限，训练只决定多快到达上限**

### 7.2 下一步方向

- **换 LSTM/GRU**：门控机制解决梯度消失，预期 perplexity 可降到 10 以下
- **多层 RNN**：增加模型容量
- **Transformer**：彻底解决长距离依赖问题
