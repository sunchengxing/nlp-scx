# 文本生成任务规划

## 任务说明
- 任务类型：语言模型（Language Model），给前N个词预测下一个词
- 数据集：wikitext-2（HuggingFace：`Salesforce/wikitext`）
- 与情感分类的区别：输入序列 → 输出序列（每个位置预测下一个词）

## 模型系列
| 阶段 | 模型 | 状态 |
|------|------|------|
| 1 | RNN 文本生成 | ⬜ |
| 2 | LSTM 文本生成 | ⬜ |
| 3 | GRU 文本生成 | ⬜ |
| 4 | Attention + LSTM 文本生成 | ⬜ |
| 5 | Transformer 文本生成 | ⬜ |

## 代码结构（复用情感分类）
```
Config          → 超参数配置
Dataset         → wikitext 数据处理
DataLoader      → 批次加载
Model           → 各模型定义
Train           → 训练 + 验证循环
Inference       → 文本生成推理
```

## 任务形式
```
输入：["The", "quick", "brown"]
目标：["quick", "brown", "fox"]   ← 每个位置预测下一个词

loss = CrossEntropyLoss(output[:-1], input[1:])
```

## 生成策略
- Greedy：每次取概率最高的词
- Top-k Sampling：从概率最高的k个词里随机采样
- Temperature：控制生成多样性

## 情感分类阶段回顾（基础对比）

| 模型 | Val Acc | 备注 |
|------|---------|------|
| FC 全连接 | 86.00% | mean pooling，丢失词序 |
| RNN | 67.28% | 双向，梯度消失严重 |
| LSTM | 83.26% | 双向，解决梯度消失 |
| GRU | 85.96% | 双向，比 LSTM 略好 |
| BiLSTM + Additive Attention | 86.80% | 最高，加权聚合 |
| BiLSTM + Multi-Head Attention | 86.72% | 与加性持平 |
| Transformer Encoder | 84.54% | 3轮最高，之后过拟合 |

**结论：**
- IMDB 这类短序列任务，BiLSTM+Attention 已足够
- Transformer 在数据量小时容易过拟合，需要更多正则化
- 词表大小对效果影响很大（5000 → 20000 有明显提升）

## 文本生成与情感分类的核心差异

| 维度 | 情感分类 | 文本生成 |
|------|----------|---------|
| 任务类型 | Sequence → Label | Sequence → Sequence |
| 输出 | 1 个类别 | 每个位置 1 个词 |
| Loss | CrossEntropy(batch) | CrossEntropy(每个位置) |
| 推理方式 | 前向一次 | 自回归逐词生成 |
| Decoder | 不需要 | 需要（Causal Mask） |
| 评估指标 | Accuracy | Perplexity（PPL） |

### Perplexity 计算
```
PPL = exp(avg_loss)
```
- loss 是 CrossEntropyLoss 的均值（按 token 算）
- PPL 越低越好，随机猜测 PPL = 词表大小

## 数据处理关键点

### wikitext-2 数据格式
```
# 原始文本（已分词，空格分隔）
" = Valkyria Chronicles III = \n"
"The game was developed by Sega ..."
```

### 语言模型的 input/target 构造
```
sequence = [t0, t1, t2, t3, t4]
input    = [t0, t1, t2, t3]      # [:-1]
target   = [t1, t2, t3, t4]      # [1:]
```

### 分块策略（Block-based）
- 不按句子截断，而是把所有 token 拼成一条长序列
- 按 `seq_len` 切块：每块 `seq_len` 个 token
- 每块的 `input = block[:-1]`，`target = block[1:]`
