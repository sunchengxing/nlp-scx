# 文本情感分类手写实践总结

## 任务说明
- 数据集：IMDb 英文电影评论情感分类
- 规模：训练集 25000 条，测试集 25000 条
- 类别：正面（1）/ 负面（0）二分类
- 文本平均长度：约 230 词

## 训练配置
| 参数 | 值 |
|------|----|
| 词表大小 | 20000 |
| 序列长度 | 256 |
| Embedding 维度 | 128 |
| Hidden 维度 | 256 |
| Batch Size | 64 |
| Epochs | 10 |
| 优化器 | Adam lr=0.001 |
| 损失函数 | CrossEntropyLoss |

## 模型对比结果
| 模型 | Val Acc | 特点 |
|------|---------|------|
| FC | 86.44% | 词袋模型，丢失词序，但对情感分类足够 |
| RNN | 67.28% | 有词序感知，但长文本梯度消失严重 |
| LSTM | 83.36% | 门控机制解决长期依赖，参数最多易过拟合 |
| GRU | 85.96% | 比LSTM参数少，效果接近，略优于LSTM |
| BiLSTM+Attention | 86.80% | 仅3轮收敛，信息利用最充分 |
| Transformer | 84.28% | 3轮收敛，第2轮突破80%，标准Encoder分类头 |

## 核心结论
1. IMDb 情感分类词袋特征足够，FC baseline 表现强
2. RNN 在长文本上梯度消失是致命缺陷
3. LSTM/GRU 差距不大，GRU 更轻量
4. 词表大小影响巨大：5000→过拟合，20000→loss稳定下降
5. Attention 收敛更快，3轮即超越所有模型
6. 复杂任务（翻译/摘要）中 Attention 优势会更显著
7. Transformer 3轮达 84.28%，收敛速度快但略低于 BiLSTM+Attention，原因是 IMDb 文本较长（~230词），Transformer 的全局注意力在小数据上容易过拟合
